# Виджет для банковских операций
Проект виджета для личного кабинета клиента банка. В данный момент проект находится на стадии разработки. Сейчас в нем реализованно ограниченный набор функциональных возможностей. Но наша команда безустали трудится над данным проектом. И в скором времени мы обязательно реализуем все наши задумки и вы сможете воспользоваться данной фичей в полной мере.

## Описание проекта
Наш виджет поможет клиентам банков легко и быстро посмотреть свои последнии операции. С помощью виджета вы сможете отсортировать операции по дате, статусу операции и другим критериям. 
 На выходе будет отображася следующая информацию:
- Дата операции
- Название платежной системы
- Шифрованный номер карты или счета, с которых производилась  операция
- Статус операции
- и т.д.

## Примеры работы виджета
На примере работы некоторых функций мы покажем какую информацию использует виджет и в каком виде он будет её выводить. 
1. Функция `get_date(date: str) -> str` вывода даты совершенной операции.
- `get_date(date: str) -> str` принимаект на вход текст в формате `"2024-03-11T02:26:18.671407"` и возвращает дату в следующем виде `11.03.2024` 

2. Функция `mask_account_card(card: str) -> str` вывода названия платежной системы или счета и зашифрованного номера.
- `mask_account_card(card: str) -> str:` может принимать на вход текст в формате `"Visa Platinum 7000792289606361"` или `"Счет 73654108430135874305"` и на выходе получить:
    - для карты `"Visa "Platinum 7000 79** **** 6361"`
    - для счета `"Счет **4305"`

## Тестирование
Основным типом тестирования является модульное тестирование и интеграционный тип.
Основным инструментом для проверки правльности работы всех функций в нашем проекте является - pytest. 
Установить его можно с помощью команды `poetry add --group dev pytest`.
Для анализа покрытия кода мы используем pytest-cov. 
Установить его можно с помощью команды `poetry add --group dev pytest-cov`. 
На данном этапе достигнуто 100% покрытие кода тестами. 
Отчет хранится в файле  `index.html`
Для тестовых файлов создан отедльная директория tests\, в которой содержаться тесты к каждому модулю из директории src\, 
а так же файл conftest с фикстурами для наших тестов.
Сгенерирована папка htmlcov, в которой хранится файл index.html с отчетом о покрытии.
